# Layer 4 — Tableau Build Guide (Finance Review Dashboard)

Build a 4-panel finance review dashboard in **Tableau Public** (free) from the CSV extracts
in this folder. The static target you're matching is `outputs/figures/layer4/dashboard_mockup.png`.

> ✅ **Status: published.** A v1 of this dashboard is live at
> https://public.tableau.com/app/profile/leo.wan3084/viz/AmazonFinanceReviewDashboardQ32022/Dashboard1
> — referenced from the README Layer 4 section. The steps below remain the reproduction
> recipe for rebuilding it from scratch.
>
> **v1 → v2 polish items** (so-what captions, scatter labels, inline %-labels, hide
> internal legends): see [`LAYER4_REFINEMENTS.md`](LAYER4_REFINEMENTS.md).

> Every number here already traces back to Layers 1–3 — no new analysis. The four extracts
> are regenerated by `src/build_tableau_extracts.py`.

**Dashboard title:** Revenue Concentration, Revenue-at-Risk & Growth Allocation
**Subtitle:** Finance Review Dashboard · Q3 2022 Snapshot · 2,846 U.S. households (consenting panel)

**Finance-clean palette (hex):** primary `#1f4e79`, accent/highlight `#c0392b`,
neutral/reference `#7f8c8d`, light grid `#cccccc`, text `#2c3e50`.
Sheet background white, no gridlines except a light y-axis, dark labels, no chart junk.

## 0. Connect the data
Tableau Public Desktop → **Connect → Text file** → add each CSV as its own data source:
`lorenz_points.csv`, `decile_contribution.csv`, `demographic_overindex.csv`,
`rar_by_decile.csv`, `category_scale_growth.csv`.

---

## Sheet 1 — "Concentration" (Lorenz curve)  → top-left
Data source: **lorenz_points.csv**
1. Columns: `cum_population_pct` → right-click → **Continuous** (Dimension).
2. Rows: `cum_gmv_pct` → **Continuous** (Dimension). Marks card → **Line**, color `#1f4e79`.
3. Drag `equality_pct` onto Rows (right of cum_gmv_pct) → it makes a 2nd axis → right-click the
   axis → **Dual Axis** → right-click → **Synchronize Axis**. Set the equality line color
   `#7f8c8d`, dashed, thinner.
4. Both axes: fix range 0–100. Titles: x "Cumulative % of households", y "Cumulative % of GMV".
5. Add an **Annotation → Point** near the top-right: *"Top decile = 36.0% of GMV · Gini 0.529."*
6. **So-what caption (text object on dashboard):** *Concentration sits at the top — but the
   long tail still matters (top 20% = 55%, not 80%).*

## Sheet 2 — "Over-Index" (demographic over-index)  → top-right
Data source: **demographic_overindex.csv**
1. Filters: drag `significant` → keep **True**; drag `included_in_report` → keep **True**.
2. Rows: `label`. Columns: `over_index_pct` (SUM is fine — one row per label).
3. Marks → **Bar**. Sort `label` **descending by `over_index_pct`**.
4. Color: create calc field **`Top driver`** = `[over_index_pct] = WINDOW_MAX([over_index_pct])`
   → drag to Color (True = `#c0392b`, False = `#1f4e79`). Or just color all `#1f4e79` and
   manually highlight the top bar.
5. Label: `over_index_pct`, format `+#,##0"%"`. Axis title: "Top-decile over-index vs panel (%)".
6. Tidy long labels (e.g. rename "order_freq: More than 10 times per month" → ">10 orders/month",
   "hh_income: $150,000 or more" → "income: $150K+") via **aliases** on the `label` field.
7. **So-what:** *Heavy cadence (>10×/mo) over-indexes +373% — engagement, not affluence,
   defines the top decile.*

## Sheet 3 — "RaR by Decile" (revenue-at-risk ladder)  → bottom-left
Data source: **rar_by_decile.csv**
1. Columns: `decile` → **Discrete** (Dimension), sorted 1→10.
2. Rows: `panel_share_pct` (SUM). Marks → **Bar**, color `#1f4e79`.
3. Highlight calc **`Mid-tier`** = `[decile] >= 6 AND [decile] <= 9` → Color (True `#c0392b`).
4. Tooltip: add `rar_dollars` ($) and `mean_prob_dropoff_pct` (P(Q3 drop-off)). Axis title:
   "% of panel revenue-at-risk"; x "GMV decile (1 = top spenders)".
5. Annotation: *"Mid-deciles 6–9 = 64% of RaR."*
6. **So-what:** *Revenue is concentrated at the top, but risk is in the middle — 64% of RaR
   on 13% of GMV.*  *(Note: outcome is single-quarter "Q3 drop-off," not churn.)*

## Sheet 4 — "Scale × Growth" (allocation matrix)  → bottom-right
Data source: **category_scale_growth.csv**
1. Columns: `scale_2022_gmv` → **Continuous**. Rows: `cagr_pct` → **Continuous**.
   Marks → **Circle** (disaggregate: Analysis → uncheck *Aggregate Measures*).
2. Detail/Label: `super_category`. Size: `n_households`. Color: `quadrant`
   (INVEST `#1f4e79`, BET-small `#2e86c1`, MAINTAIN `#7f8c8d`, HARVEST `#c0392b`, Boundary `#bdc3c7`).
3. Reference lines: Analytics → **Constant Line** on x at value = `median_scale` (369093),
   on y at value = `median_cagr_pct` (24.8); color `#7f8c8d`, dashed, label "median".
4. Axis titles: x "2022 GMV scale ($)", y "4-year CAGR (%)". Format x as `$#,##0`.
5. **So-what:** *Home & H&PC are the cleanest INVEST — high growth that new customers
   actually enter through.*

---

## Assemble the dashboard
1. New **Dashboard**, size **1300 × 950** (or Automatic). Untitled → add a **Text** title object:
   the dashboard title (16pt bold `#1f4e79`) + subtitle line (`#7f8c8d`).
2. Drag a **Horizontal** container, then two **Vertical** containers inside → drop the 4 sheets
   into a 2×2 grid (TL Concentration, TR Over-Index, BL RaR, BR Scale×Growth).
3. Under each sheet add a small **Text** object with its so-what caption (9–10pt italic `#555`).
4. Hide redundant legends/titles for a clean look; keep the quadrant color legend on Sheet 4.

## Publish + link  ✅ done for v1

1. **File → Save to Tableau Public** (needs a free Tableau Public account) → save the workbook.
   *(v1 published as `AmazonFinanceReviewDashboardQ32022/Dashboard1`.)*
2. Copy the public URL → live link is now wired into the README Layer 4 section
   ([URL](https://public.tableau.com/app/profile/leo.wan3084/viz/AmazonFinanceReviewDashboardQ32022/Dashboard1)).
3. *(Optional, not yet done)* Record a short **GIF** (~8–10s) hovering/filtering — e.g. with
   Kap (mac) or ScreenToGif (win) — save to `outputs/figures/layer4/dashboard.gif` and embed
   it under the static mockup. Useful but not required; the live link already conveys interactivity.

The static mockup stays in the README as the preview; the live link is the upgrade. Any
future iteration of the dashboard re-publishes over the same URL.
